# BOM Change Taxonomy by Industry

## How to read this file

* This is a cross-industry, impact-based taxonomy for BOM-related change control.
* "BOM properties" includes BOM-adjacent configuration, sourcing, effectivity, compliance, and approval fields because many real changes are triggered by source, site, process, effectivity, or qualification changes even when the component list itself does not change.
* "Approving roles" are normalized role names. Actual companies use different titles, but the roles are grounded in public standards and supplier/customer change-control examples.
* Impact classes:
  * A = Critical / major / Class I
  * B = Significant / controlled
  * C = Minor / Class II
  * D = Temporary deviation / waiver / concession
  * E = Emergency / urgent

## Working assumptions

* New part number versus revision-only decisions are not universal. They depend on interchangeability policy, contract terms, service strategy, and customer rules.
* The approval-role column is intentionally critical and conservative. In regulated or customer-controlled environments, it is safer to include the external design authority or customer quality representative when there is any real doubt.
* For construction, the term "BOM" is broadened to reflect the way the industry actually controls change: model objects, BoQ items, submittals, specifications, fabrication details, cost codes, and schedule effects.
