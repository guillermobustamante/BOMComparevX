# Codex Skill Order

Use these skills in order unless a task clearly requires only one layer:

1. ui-foundation
2. ui-generation
3. frontend-implementation
4. ui-qa

Do not start frontend coding for a major new module until ui-foundation and ui-generation have passed QA.
Do not treat frontend-implementation as a UI strategy skill.
Use ui-qa as a hard quality gate before acceptance.

Additional project skill:

- backlog-execution
  Use for end-to-end backlog or story delivery inside this repo when the task requires docs-first discovery, conservative code changes, repo-aligned validation, and runtime QA before claiming completion.
